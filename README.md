> A modern, responsive feedback collection form built with Next.js and Supabase

## Features

- **Simple Feedback Collection**: Collect user feedback with name (optional), 1-5 star rating, and message
- **Real-time Validation**: Client-side form validation with loading states
- **Supabase Integration**: Secure backend storage with Row Level Security
- **Responsive Design**: Beautiful Tailwind CSS styling that works on all devices
- **Success Feedback**: User-friendly success message with option to submit another response
- **Error Handling**: Graceful error handling with user-friendly messages

## Quick Start

### Prerequisites

- Node.js 20+ installed
- A Supabase account (free tier available at [supabase.com](https://supabase.com))

### Installation

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd feedback-app
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Set up Supabase (see Configuration section below)

4. Run the development server:
   ```bash
   npm run dev
   ```

5. Open [http://localhost:3000](http://localhost:3000) in your browser

## Configuration

### Supabase Setup

1. Create a new project at [Supabase.com](https://supabase.com)

2. Go to the **SQL Editor** and run this query to create the feedback table:
   ```sql
   create table feedback (
     id bigint generated by default as identity primary key,
     created_at timestamp with time zone default timezone('utc'::text, now()) not null,
     name text,
     rating integer,
     message text
   );
   ```

3. Enable Row Level Security (RLS):
   - Go to **Authentication** → **Policies**
   - Create a new policy for the `feedback` table
   - Choose "Enable insert access for all users" (or "anon" role)
   - This allows anonymous users to submit feedback without authentication

### Environment Variables

Create a `.env.local` file in the root directory:

```env
NEXT_PUBLIC_SUPABASE_URL=your-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

You can find these values in your Supabase project settings under **API**.

## Usage

### Submitting Feedback

1. Navigate to the home page
2. Optionally enter your name
3. Select a rating (1-5 stars) by clicking the star buttons
4. Enter your feedback message (required)
5. Click "Submit Feedback"
6. You'll see a success message when the feedback is submitted

### Viewing Feedback Data

To view submitted feedback:
1. Go to your Supabase dashboard
2. Navigate to **Table Editor**
3. Select the `feedback` table
4. View all submitted feedback entries with timestamps

## Architecture

### Tech Stack

- **Frontend**: Next.js 16 with App Router, React 19, TypeScript
- **Styling**: Tailwind CSS 4
- **Backend**: Next.js API Routes
- **Database**: Supabase (PostgreSQL)
- **Compiler**: React Compiler (experimental), Babel

### Project Structure

```
feedback-app/
├── app/
│   ├── api/
│   │   └── feedback/
│   │       └── route.ts          # API endpoint for feedback submission
│   └── page.tsx                  # Home page with form
├── components/
│   └── FeedbackForm.tsx          # Main feedback form component
├── lib/
│   └── supabaseClient.ts         # Supabase client configuration
├── .env.local                    # Environment variables (not in git)
└── package.json                  # Dependencies and scripts
```

### API Endpoints

**POST /api/feedback**
- Accepts: `{ name?: string, rating: number, message: string }`
- Returns: `{ success: true }` or `{ error: string }`
- Validates Supabase credentials and inserts feedback into database
